# PDF 自动裁切（画面增强）原理与方法

本文档说明 PdfReader 中「自动裁切」功能的实现原理、算法步骤、对齐策略，以及它与手动裁切的关系。对应代码位于 `PdfReaderActivity.kt`：`calculateSmartCrop()`（单页内容检测）、`calibrateVerticalBand()` / `launchVerticalBandCalibration()`（垂直对齐基准带标定）、以及 `renderPageSync()` 中的裁切应用逻辑。

---

## 一、自动裁切解决什么问题

扫描版 / 导出版 PDF 通常带有大量白边，以及页眉、页脚、页码、留白等「页面装饰」。这些区域对阅读没有价值，却白白占用了屏幕空间，导致正文偏小。

自动裁切的目标：**自动检测出每页真正有文字/图片内容的矩形区域，把四周的空白与装饰裁掉，让正文尽量铺满屏幕**。

- 开关位置：阅读页设置 → 「外观」→ 「画面增强」区块里的 **自动裁切** 开关。
- 注意：该开关受 VIP 限制（`PaymentManager.hasVipAccess`），未解锁时开启会被拦截并弹出购买提示。
- 开启后会清空裁切缓存并触发一次「垂直对齐基准带」标定，随后页面按新规则重新渲染。

## 二、单页内容检测（核心算法）

核心函数是 `calculateSmartCrop(page: PdfRenderer.Page): RectF`，返回一个归一化矩形 `(left, top, right, bottom)`，取值范围 0~1，表示内容区域占整页的比例。

### 步骤拆解

**1. 渲染低分辨率缩略图**

不直接处理原图（几 MB / 几千像素），而是先把整页渲染成宽 **200px** 的缩略图：

```
thumbWidth = 200
thumbHeight = 200 / page.width * page.height
```

200px 足够看清「宏观排版」（哪里有内容、哪里是空白），又能把内存和 CPU 消耗降到极低。背景先填白，避免透明底 PDF 误判。

**2. 像素提取**

把缩略图像素读入 `IntArray`，准备做投影统计。

**3. 灰度 + 二值化投影（Projection）**

遍历每个像素，只要它「不那么白」就计入内容：

```
不是纯白判定：R < 245 或 G < 245 或 B < 245
```

对每一行、每一列分别统计「内容像素个数」，得到两个数组：

- `rowCounts[y]`：第 y 行有多少内容像素
- `colCounts[x]`：第 x 列有多少内容像素

这一步的本质是**把二维版面压成一维投影**：有内容的行/列计数高，纯白边行/列计数为 0。

**4. 动态噪点阈值 + 求内容包围盒**

设定噪点阈值（防止偶发灰尘、极细扫描黑边误判）：

```
noiseThresholdX = 缩略图宽 * 1.5%
noiseThresholdY = 缩略图高 * 1.5%
```

然后从四个方向向中心收缩，跳过计数低于阈值的行/列：

```
top    从 0 向下，直到 rowCounts[top]  > 阈值
bottom 从底向上，直到 rowCounts[bottom] > 阈值
left   从 0 向右，直到 colCounts[left]  > 阈值
right  从右向左，直到 colCounts[right] > 阈值
```

得到的 `(left, top)` ~ `(right, bottom)` 就是内容包围盒（在缩略图坐标系）。

**5. 安全边距**

为避免边缘的衬线字体笔画、图片边框被微微切断，向外扩 **2%** 边距：

```
padX = 缩略图宽 * 2%
padY = 缩略图高 * 2%
left  = max(0, left  - padX)
top   = max(0, top   - padY)
right = min(W-1, right + padX)
bottom= min(H-1, bottom+ padY)
```

**6. 兜底（空白页）**

若某页完全空白（`left >= right` 或 `top >= bottom`），直接返回整页 `RectF(0,0,1,1)`，即「不裁切」，避免把白页裁没。

最终把缩略图坐标换算回 0~1 归一化比例返回。

---

## 三、水平方向：逐页独立裁切

水平方向（左右白边、页眉页脚宽度差异）**每页单独计算**，因为：

- 插图书页、图表页的左右留白和文字页不同；
- 奇偶页、章节首页的页眉位置可能不一致。

逐页结果用 `LruCache<Int, RectF>(100)`（`autoCropCache`）缓存，避免每帧重算。渲染时：

```
transX = cachedRect.left          // 水平偏移
contentWidthRatio = cachedRect.width()   // 内容宽度占比
```

---

## 四、垂直方向：统一对齐基准带

如果每页都按自己的上下边独立裁切，翻页时正文会**上下跳动**（因为每页内容的垂直位置略有差异），阅读体验差。

为此引入「垂直对齐基准带」标定（`calibrateVerticalBand()`）：

1. 在全文档**均匀采样最多 40 页**（按 `i * (pageCount-1) / (sampleCount-1)` 取索引），逐页算内容框的 `top`、`bottom`。
2. 把收集到的所有 `top`、`bottom` 分别排序，取**中位数**作为统一基准：
   ```
   autoCropBandTop    = median(所有页的 top)
   autoCropBandBottom = median(所有页的 bottom)
   ```
   中位数比平均值更稳健，能扛住偶尔出现的满版插图页 / 异常页。
3. 标定完成后清空裁切缓存并重新渲染，使后续页面按基准带对齐。

**渲染时的对齐规则**（`renderPageSync` 内）：

```
若已标定基准带，且 本页内容框完全落在基准带内：
    transY = autoCropBandTop
    contentHeightRatio = bandBottom - bandTop      // 用统一基准，上下对齐
否则：
    transY = cachedRect.top
    contentHeightRatio = cachedRect.height()       // 用本页自己的裁切（不强行对齐）
```

「本页内容框完全落在基准带内」这一判断（容差 0.01）是为了**保护满版图片 / 插图**：这类页的内容本身就顶天立地，不该被基准带把上下裁掉，所以回退到逐页裁切。

## 五、裁切如何应用到渲染

得到 `transX / transY / contentWidthRatio / contentHeightRatio` 后，在 `renderPageSync` 中构造变换矩阵：

```
contentPdfWidth  = page.width  * contentWidthRatio
contentPdfHeight = page.height * contentHeightRatio
scale = renderWidth / contentPdfWidth          // 按屏幕宽度铺满
renderHeight = contentPdfHeight * scale

transform.postTranslate(-page.width * transX, -page.height * transY)  // 平移到内容原点
transform.postScale(scale, scale)                                     // 缩放到铺满
```

即：先把坐标原点移到内容区左上角，再整体缩放，使内容宽度正好等于屏幕宽度，从而在屏幕上最大化显示正文。

> 若「自动裁切」关闭，则使用**手动裁切**值（`cropLeft/right/top/bottom`，默认全 0）→`transX=cropLeft, contentWidthRatio = 1 - cropLeft - cropRight`，依此类推。手动与自动互斥：开了自动就以自动结果为准。

## 六、与「手动页面裁切」的关系

「外观」里除了自动裁切开关，还有 **页面裁切** 四个独立滑块：左 / 右 / 上 / 下。

- 四个值持久化在书籍实体 `PdfBook` 的 `cropLeft / cropRight / cropTop / cropBottom`（Room v6→v7 迁移时由旧的对称 `cropMargin` 拆出）。
- 这是**逐本书、用户主动设定**的硬性裁切，优先级最高：渲染时先按这四个值计算初始 `transX/transY/contentWidthRatio/contentHeightRatio`。
- 仅当「自动裁切」开启时，才会用 `calculateSmartCrop` 的结果**覆盖**上述计算（水平逐页 + 垂直基准带）。
- 想精确控制某一本书的版心，关掉自动裁切、手动拉四个滑块即可。

---

## 七、性能与边界说明

- 单页检测只在 **200px 缩略图** 上做，开销极小；结果进 `LruCache` 避免重复计算。
- 基准带标定在 `Dispatchers.IO` 异步进行，逐页 `yield()` 让出，避免长期占用渲染锁阻塞翻页；带 `autoCropCalibrating` 重入保护。
- 阈值选择（白判定 245、噪点 1.5%、安全边 2%）是经验值，兼顾「切得干净」与「不切到内容」。
- 异常（某页渲染失败等）一律回退到 `RectF(0,0,1,1)`（原图不裁），保证不会因裁切把页面弄没。
