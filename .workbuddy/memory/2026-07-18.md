### 关键规格（用 onnxruntime Python 参照实现逐行验证）
- rec 输出 `[1,T,6625]`（**[1,T,C]，非 [1,C,T]**），C=6625=blank+6623字+空格。字典长度必须严格=6625，否则 `t=len~/c` 错位、整段乱码。
- det 输入 `[1,3,H,W]`，H/W **必须为 32 的整数倍**（DB 上/下采样拼接约束），否则 onnxruntime 在 Add 节点报 broadcast 错误崩溃。
- DB 输出是收缩的文字核心，需 **unclip 外扩**（dist=w*h*ratio/(2*(w+h))，ratio 1.5~2.0 均可）才能裁出完整文字；不外扩则条带过窄乱码。

## OCR「又卡又一个字都不准」根因修复（重要）
- **根因**：`pdf_ocr_service.dart` 的 `_preprocessDet`/`_preprocessRec` 用 HWC 交错布局 `(y*W+x)*3+c` 填充 Float32List，但 `OrtValue.fromList(input,[1,3,H,W])` 声明的是 NCHW 平面布局。onnxruntime 把交错像素当分离通道读，输入被打乱 → 检测框错位、识别全乱码。之前 Python 参考验证做了 `np.transpose(2,0,1)`（CHW），掩盖了这个 Dart 专属 bug。
- **诊断方法（可复用）**：用 `~/.workbuddy/binaries/python/envs/default/bin/python` + onnxruntime + Pillow，PIL 造多行中文页 → 完整复刻 Dart 全流程（det 预处理 → sigmoid 概率图 → 4 邻接 BFS 连通域 → unclip 扩框 → 裁剪 → rec → CTC）。清晰文字下 ImageNet/Paddle 归一化、BGR/RGB 四种组合都能识别对 → 排除归一化；改回真实多行页复刻发现只要 transpose 成 CHW 就全对，定位到布局。
- **修复**：两个预处理改 NCHW 平面 `buf[c*plane + y*W + x]`，通道 BGR；新增 rec 专用归一化常量 `_recNormMean/_recNormStd=0.5`，rec 用 `(x/255-0.5)/0.5`，det 保持 ImageNet。已 analyze 0 error。
- **模型确认**：PP-OCRv4，det=ch_PP-OCRv4_det_infer.onnx（输出 sigmoid_0.tmp_0 [N,1,H,W]），rec=ch_PP-OCRv4_rec_infer.onnx（输出 softmax_11.tmp_0 [N,T,6625]），dict 6625 类（blank@0，空格@6624）。均来自 HuggingFace SWHL/RapidOCR。
- **待跟进**：真机重跑验证；「卡」是 root-isolate ffi 固有，如仍慢可考虑降 det 长边(960)/降渲染分辨率。连通域法在行距极小的版式可能并行合并成大块（当前测试未出现），必要时再换 minAreaRect+pyclipper。

## 重排质量重构：图片块检测改「行带整块裁剪」+ 缓存 v2（Phase B）
- 用户反馈：重排「一堆碎截图、字没几个、看不清」；且「打开书就 OCR」不对（后者此前已由 `BookViewerPage.shouldAutoStartOcrOnOpen()=false` 修复，本轮确认无需再改）。
- 根因：`pdf_ocr_service.detectImageBlocks` 旧版按「非白非文本像素连通域」找图块——图表内部白缝把一幅图碎成几十个小块（=一堆截图），DB 漏检的正文行也被误判成图块吞掉正文。
- 修复：`detectImageBlocks` 重写为**行带(row-band)**三步——逐行判「图形行」(墨点够多 / 文本占墨点比≤0.25 / 非正文行) → 连续图形行(≤3行空白间隙)聚成带 → 仅留高≥页高6%、宽≥页宽12%的带，按带内墨点横向范围整块裁**一张**图。效果：一图=一截图；纯文字页 0 图块，全走 OCR 段落重排(ePub 式)。阈值：inkRowMin=0.015 / figureTextRatioMax=0.25 / textRowMin=0.010 / minBandHeightFrac=0.06 / minBandWidthFrac=0.12 / rowGapTol=3。
- `pdf_ocr_cache_service` 缓存文件名 v1→v2，令旧碎裂缓存失效、重排按新算法重跑。
- 图放在对应文字下方：`pdf_ocr_reader_view._mergeFlow` 已按 top 坐标把图块内联到其上方文字段落之后，无需改。
- 验证：`flutter analyze` 两文件 0 error。CHANGELOG + 依赖文档已同步。
