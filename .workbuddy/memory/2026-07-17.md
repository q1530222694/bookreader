# 2026-07-17 工作记录

## 修复 bookreader Android debug 启动失败（两连错）

### 错误 1：Java home invalid
- 现象：`org.gradle.java.home` 写死 Windows 路径 `C:/Program Files/Android/Android Studio/jbr`，macOS 上无效，Gradle 启动即 FAILURE。
- 修复：`android/gradle.properties` 改为 macOS AS JBR 路径 `/Applications/Android Studio.app/Contents/jbr/Contents/Home`（Java 21.0.10）。
- 注意：**不能直接删该行**——本机 PATH 的 `java` 是 Corretto-25（Java 25），Gradle 8.10.2/9.1 不支持 Java 25，会落到 25 而失败。必须显式指向 Java 17~21。

### 错误 2：Gradle 版本错配 AGP
- 现象：AGP `com.android.application` 9.0.1（settings.gradle.kts）要求 Gradle ≥ 9.1.0，但 wrapper 锁在 8.10.2 → `assembleDebug` FAILURE。
- 修复：`android/gradle/wrapper/gradle-wrapper.properties` 从 `gradle-8.10.2-all.zip` 改为 `gradle-9.3.1-all.zip`。
- 选型依据（Flutter 3.44.4 源码 gradle_utils.dart）：`templateDefaultGradleVersion=9.1.0`，`maxKnownAndSupportedGradleVersion=9.3.1`。选 9.3.1 是因为它支持本机 Java 25 的启动 JVM，且 ≥9.1.0 满足 AGP。

### 验证
- `./gradlew --version` 输出 `Daemon JVM: '/Applications/Android Studio.app/Contents/jbr/Contents/Home' (from org.gradle.java.home)` + `Gradle 9.3.1`，确认两处修复生效。
